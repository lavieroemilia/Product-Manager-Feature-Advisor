# Features

Cada carpeta aquí documenta una feature aplicando el framework. La carpeta `template/` tiene el brief en blanco para empezar una nueva.

---

## Cómo crear una feature nueva

1. Copia la carpeta `template/` y renómbrala:
   ```
   YYYY-MM-nombre-corto-de-la-feature
   ```
   Ejemplo: `2024-11-onboarding-checkout`

2. Renombra el archivo dentro a `feature-brief.md`

3. Rellena cada fase en orden. Si no puedes responder una sección, ese es el trabajo de discovery que falta — no lo saltes.

## Estados posibles

- **Discovery** — Investigando el problema, sin solución definida aún
- **Shaping** — Oportunidad validada, definiendo el pitch
- **En desarrollo** — El equipo está construyendo
- **Lanzada** — En producción, midiendo el outcome
- **Descartada** — Se decidió no construir (con razonamiento documentado)
