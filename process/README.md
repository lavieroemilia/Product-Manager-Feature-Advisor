## Principios

**Outcomes sobre outputs**  
El trabajo no es entregar features. Es cambiar comportamientos de usuarios de forma que impulsen resultados de negocio.

**Discovery continuo**  
Las oportunidades vienen de conversaciones reales con usuarios, no de suposiciones. El hábito de entrevistar regularmente es la base de todo lo demás.

**Testea supuestos, no ideas**  
Antes de construir, identifica el supuesto más arriesgado de tu solución y ponlo a prueba.

**Shape antes de entregar**  
El equipo no debería recibir una lista de requisitos ni un problema sin forma. Recibe un pitch: problema claro, solución con límites definidos y appetite acordado.

**Decide con evidencia, documenta el razonamiento**  
Una decisión tomada con la información disponible en ese momento es una buena decisión, aunque el resultado sea distinto al esperado.
